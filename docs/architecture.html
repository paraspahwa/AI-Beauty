Payment Flow
Razorpay — one-time unlock + Studio Pro subscription
📄 One-Time Report Unlock
₹299 India$3.99 Intl
Permanent unlock for a single report. Triggered by Paywall component.
⭐ Studio Pro Subscription
₹999/mo India$12.99/mo Intl
150 AI gens/month, wardrobe capsule, ingredient scanner, assets vault.
Payment Flow Steps
1
POST /api/payments/create — currency from CF-IPCountry/X-Vercel-IP-Country header → Razorpay order/subscription created
2
Client: Razorpay Checkout modal opens with order_id
3
POST /api/payments/verify — HMAC-SHA256 signature check with timingSafeEqual → updates payment row status (does NOT unlock report)
4
Razorpay sends payment.captured webhook → POST /api/webhooks/razorpay
5
Webhook: HMAC verify → record_webhook_event() (idempotency) → complete_webhook_payment() RPC → atomically unlocks report
6
Client polls or realtime → report unlocked, Paywall removed

Security Rules
Critical invariants that must never be violated
🔴createSupabaseAdminClient() bypasses RLS — always validate user_id ownership explicitly before any query
🔴selfies Storage bucket must remain private — access only via signed URLs, never public
🔴INTERNAL_API_SECRET must be ≥16 chars — /api/internal/trigger-visuals returns 503 otherwise
🔴HMAC comparisons always use timingSafeEqual (crypto) — never string equality ===
🔴Currency derived from CF-IPCountry / X-Vercel-IP-Country server headers only — never trust client hint
🔴Webhook is the ONLY authoritative unlock — POST /api/payments/verify must never set is_paid
🔴PAYMENT_TEST_ALLOW_IN_PROD must be false in production
🔴Supabase Site URL must be set to production domain in Dashboard → Auth → URL Configuration
🔴ADMIN_EMAIL_ALLOWLIST — never hardcode email addresses in source code
🔴Replicate SDK: always useFileOutput: false — FileOutput objects break URL validation
