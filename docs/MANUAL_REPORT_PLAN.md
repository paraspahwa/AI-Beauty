# Manual Report Builder — Product Plan

> **Feature:** Let users create a personalised style report **without uploading a selfie** by answering a guided questionnaire about their own features.
>
> **Why:** Privacy-conscious users, users without a good selfie, or users who want to validate/override the AI's reading. Also opens a **free discovery path** that converts to paid.

---

## 🎯 What We're Building

A **multi-step guided form** at `/build` (or `/quiz`) where the user self-declares:
- Face shape
- Skin tone & undertone
- Hair type & current colour
- Eye colour & shape
- Style goals

...and receives a **full styled report** generated by GPT — no selfie, no AI vision, no AWS Rekognition.

**The report output is identical in structure** to the AI-analysed report. Same tabs, same cards, same PDF. The only difference is the data source.

---

## 🗺️ User Flow

```
/build
  Step 1: Face Shape      → Visual picker (oval / round / square / heart / diamond)
  Step 2: Skin Tone       → Fitzpatrick scale slider + undertone selector (warm/cool/neutral)
  Step 3: Hair            → Type (straight/wavy/curly/coily) + current colour swatch
  Step 4: Eyes            → Colour picker + shape selector
  Step 5: Style Goals     → Multi-select tags (minimal, bold, classic, trendy, etc.)
  Step 6: Review          → Summary card + "Generate My Report" CTA
		  ↓
  POST /api/manual-report/generate
		  ↓
  Redirect → /report/[id]   (same report page, `source: "manual"`)
```

### New Pages

| Route | Purpose |

### New API Route

**`POST /api/manual-report/generate`**

```ts
Body: {
  faceShape:   FaceShape           // "Oval" | "Round" | ...
  skinTone:    string              // e.g. "Medium Warm"
  undertone:   "Warm"|"Cool"|"Neutral"
  hairType:    string              // "Wavy"
  hairColor:   string              // "#8B5A2B" hex
  eyeColor:    string              // "Brown"
  eyeShape:    string              // "Almond"
  styleGoals:  string[]            // ["minimal", "classic"]
}

Response: { reportId: string }
```

**What it does:**
1. Auth-gate: user must be logged in (same as selfie flow)
2. Calls GPT-4o with a structured prompt that includes all the self-declared features
3. Constructs the same `ReportResult` shape the AI analysis produces
4. Inserts into `reports` table with `source: "manual"` and `image_path: null`
5. Returns `reportId` → client redirects to `/report/[id]`

### DB change (migration)

```sql
-- 0019_manual_report_source.sql
ALTER TABLE reports ADD COLUMN IF NOT EXISTS source TEXT DEFAULT 'ai' CHECK (source IN ('ai', 'manual'));
```

No other schema changes — the report row shape is identical.

### GPT Prompt Template

```
You are a professional beauty and style analyst.
The user has self-declared their features. Generate a complete, personalised style report.

Features:
- Face shape: {{faceShape}}
- Skin tone: {{skinTone}} (undertone: {{undertone}})
- Hair type: {{hairType}}, current colour: {{hairColor}}
- Eye colour: {{eyeColor}}, eye shape: {{eyeShape}}
- Style goals: {{styleGoals}}

Return a JSON object matching the ReportResult schema:
{
  faceShape: { shape, traits, confidence: 0.95 },
  colorAnalysis: { season, undertone, palette, metals, avoidColors },
  skinAnalysis: { type, concerns, zones, routine },
  spectacles: { recommended, avoid, tips },
  hairstyle: { styles, lengths, colors, hairType, stylingTips },
  featureBreakdown: { eyes, eyebrows, nose, lips, cheekbones, jawline, forehead }
}
```

## 💰 Monetisation

The manual report follows the **same paywall** as the AI report:
- Free: face shape + style introduction
- Paid ($9.99): all sections + PDF

**Potential addition:** Offer the manual report at a **lower price point ($4.99)** since there's no AI vision cost — this captures price-sensitive users.

| Plan | AI Report | Manual Report |
|---|---|---|

## ⚠️ Risks & Assumptions

| Risk | Mitigation |
|---|---|
| GPT output doesn't match `ReportResult` schema exactly | Use Zod validation + retry with clarification prompt |
| Users self-declare incorrectly (wrong face shape) → bad recommendations | Add a "Help me identify" tooltip with visual guides |
| Manual report cannibalises AI report revenue | Price it lower + upsell: "Upgrade to AI analysis for more accuracy" |
| `image_path: null` breaks visual generation routes | Guard every visual route with `if (!report.image_path) return 422` |
| Users expect Virtual Try-On without a selfie | Require selfie upload for Try-On even on manual reports |
