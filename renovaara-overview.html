Renovaara

AI-powered beauty analysis SaaS — complete technical reference for the entire codebase, architecture, and pipeline.

Next.js 15 React 19 TypeScript Supabase GPT-4o Razorpay Vercel Pro FAL AI Replicate

Tech Stack

Frontend

  • Next.js 15 App Router
  • React 19 + TypeScript
  • Tailwind CSS 3
  • Radix UI + class-variance-authority
  • Framer Motion animations

Backend / Database

  • Supabase (PostgreSQL + Auth + Storage)
  • Row-Level Security on all tables
  • 20 sequential SQL migrations
  • pg_cron for background jobs
  • Stored RPCs for all multi-table writes

AI Services

  • OpenAI GPT-4o (vision stages)
  • OpenAI GPT-4o-mini (text stages)
  • AWS Rekognition (face detection)
  • FAL AI (makeup, hair, try-on)
  • Replicate (SDXL, Flux Kontext, SeedDream 4.5)

Deployment

  • Vercel Pro (required for maxDuration=60)
  • Selfies bucket: private Supabase Storage
  • Edge middleware for rate-limiting + auth
  • Server-Sent Events (SSE) for pipeline stream

Key helpers

  • chatJSON<T>() — all OpenAI calls, always response_format: json_object
  • withRetry() — 2 attempts, 250ms×attempt backoff
  • compressForAI() — 512px max before sending to vision
  • blendConfidence() — 0.7×GPT + 0.3×(Rekognition/100), threshold 0.65

Models

  • env.openai.visionModel → gpt-4o
  • env.openai.miniModel → gpt-4o-mini
  • Do NOT change to GPT-5 — outputs are shaped for these models

Output validation

  • All AI output validated by Zod schemas in lib/ai/contracts.ts
  • Normalization functions safe-fallback on bad AI output
  • Every prompt MUST instruct JSON output

Currency Detection

src/lib/currency.ts — detectCurrency(): Asia/Kolkata timezone → INR, else USD. Derived from CF-IPCountry / X-Vercel-IP-Country server headers. Never trust client currency hint.

Payment Flow

⚠ Critical: The webhook is the ONLY authoritative unlock source. POST /api/payments/verify does NOT set is_paid. Only the payment.captured webhook event does (via complete_webhook_payment RPC).

Checkout flow (one-time)

  1. Client calls POST /api/payments/create → Razorpay order ID
  2. Razorpay checkout opens in browser
  3. On success, client calls POST /api/payments/verify (signature check only — no unlock)
  4. Webhook fires payment.captured → complete_webhook_payment RPC unlocks report

Subscription flow

  1. Client creates Razorpay subscription
  2. Webhook fires subscription.charged → unlocks Studio Pro
  3. Monthly renewal handled by Razorpay + webhook

Security rules

  • HMAC-SHA256 via timingSafeEqual — never use ===
  • Webhook idempotency: record_webhook_event(event_id)
  • PAYMENT_TEST_ALLOW_IN_PROD must be false in prod

Personalization Memory

File: src/lib/ai/memory.ts

persistStylePrefs()

Called after every analysis completes. Saves extracted style preferences (colors liked, styles avoided, etc.) to user_style_prefs via upsert_style_prefs RPC.

buildPersonalizedSystemBase()

Called at start of next analysis or chat. Reads user_style_prefs and injects context into the system prompt so AI remembers past preferences.

Security

🔒 Critical Rules
  • createSupabaseAdminClient() bypasses RLS — always validate user_id ownership explicitly
  • selfies storage bucket must remain private
  • INTERNAL_API_SECRET must be ≥16 chars — /api/internal/trigger-visuals returns 503 otherwise
  • Supabase Site URL must be set to production domain in Dashboard → Auth → URL Configuration
  • Webhook HMAC uses timingSafeEqual — never use === for signature comparison
  • Rate limiting: /api/analyze → 8 req/60s per IP, /api/chat → 30 req/60s per IP (in-process LRU)
  • Hard quota gate: DAILY_ANALYSIS_QUOTA = 10 per user per day in the analyze route handler
  • PAYMENT_TEST_ALLOW_IN_PROD must be false in production
  • Currency is derived from server headers only, never from client input
  • OWASP Top 10 compliance expected — no SQL injection (uses parameterised RPCs), no XSS (React escaping), no IDOR (RLS)

Middleware

File: src/middleware.ts

1. Rate Limiting (in-process LRU)

  • 4096-entry LRU, resets on cold start
  • /api/analyze → 8 req/60s per IP
  • /api/chat → 30 req/60s per IP

2. Auth Protection

Protected paths: /upload, /report, /dashboard, /success, /admin — all require Supabase session, redirect to login otherwise.

3. Session Refresh

updateSession() from lib/supabase/middleware.ts called on every request to refresh the Supabase session cookie.
